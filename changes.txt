Navigation API Data Structure & Implementation Guide
How API Data is Displayed
1. Main Navigation Structure
The navigation follows a 3-level hierarchy:
Main Menu → Sub Items → Third Level Items (with categories)
Example Flow:

Personal (Main Menu)

Individual (Sub Item)

Accounts (Category)

NBC Individual Current Account (Third Level Item)
NBC Direct Current Account (Third Level Item)
etc.







2. Data Display Pattern
Main Menu Items (MenuItem[])
typescript{
  label: "Personal",           // Display name
  hasDropdown: true,          // Shows dropdown indicator
  subItems: [...]             // Contains sub-navigation
}
Sub Menu Items (SubMenuItem[])
typescript{
  label: "Individual",
  description: "Personal banking solutions",  // Subtitle text
  hasThirdLevel: true,                       // Enables third level
  categories: {                              // Groups third level items
    "Accounts": [...],
    "Loans": [...],
    "Insurance": [...]
  },
  thirdLevelContent: {                       // Hero content for each category
    "Accounts": {
      imageUrl: "/images/...",
      description: "...",
      keyFeatures: [...],
      buttonText: "...",
      buttonLink: "..."
    }
  }
}
Third Level Items (ThirdLevelItem[])
typescript{
  label: "NBC Individual Current Account",
  href: "/personal/accounts/nbc-individual-current",
  description: "Our flagship individual current account",
  parentCategory: "Accounts",              // Links to category
  hasNested: true,                        // Optional: has sub-items
  nestedItems: [...]                      // Optional: nested navigation
}
3. Visual Display Components
Mega Menu Layout

Left Panel: Categories list
Center Panel: Items in selected category
Right Panel: Hero content with image, features, and CTA

Action Buttons
typescript{
  label: "Login",
  variant: "primary",        // Styling variant
  icon: "User",             // Icon component
  items: [...]              // Dropdown items
}

Best Practices
Naming Conventions

Main Menu: Broad categories (Personal, Business, Corporate)
Sub Menu: Specific customer segments (Individual, Private, Privileged)
Categories: Product/service types (Accounts, Loans, Cards)
Items: Specific products (NBC Individual Current Account)

URL Structure
/main-menu/sub-menu/category/item
/personal/accounts/nbc-individual-current
/business/loans/working-capital
/corporate/treasury/spot-fx
Content Requirements

Description: Brief, clear explanation
Key Features: 3-5 bullet points highlighting benefits
Images: High-quality, relevant visuals
CTAs: Action-oriented button text

Data Consistency

Always include parentCategory for proper grouping
Maintain consistent href patterns
Provide thirdLevelContent for categories with hero sections
Use descriptive altText for images


Implementation Steps

Define the structure - Decide on main menu, sub-menu, and categories
Create the data objects - Follow the TypeScript interfaces
Add routing - Ensure all href values have corresponding routes
Create content - Add images, descriptions, and features
Test navigation - Verify all links and dropdowns work
Update components - Ensure UI components handle the new structure

This structure provides a flexible, scalable navigation system that can accommodate complex banking products while maintaining a clear user experience.
