Here's a concise commit message covering all the changes:

---

**feat: migrate order detail and payment pending pages to real API data**

**Order Detail (`OrderDetailPageSection`)**
- Refactored to accept `orderId` as a prop (passed from page params) instead of reading from `useParams` internally
- Replaced mock order data with live API calls via `useCheckout` (`getOrderById`) and `usePayment` (`getPaymentsByOrder`)
- Added `mapOrderStatus` to translate API order statuses to `OrderProgressStatusI` for the progress tracker
- Added `formatDate`, `formatTime`, `formatPaymentMethod` helpers
- Replaced static "Payment Method" card with a dynamic "Payment Info" section showing real payment data, control number (with copy-to-clipboard), processing fee, timestamps, and status badges
- Replaced hardcoded shipping address fields with `shippingAddress` object from API (`fullName`, `addressLine1/2`, `city`, `state`, `country`, `phone`)
- Added "Delivery Info" section rendering `deliverySelections` (mode, status, cost per product)
- Conditionally show "Cancel order" button only for `PENDING`, `CONFIRMED`, `PROCESSING` statuses
- Removed mock driver info, delivery arrived banner, and store verified badge sections
- Added full skeleton loading state while order is fetching

**ProductItemCard**
- Converted from accepting an `OrderDetailItemI` object to individual props (`name`, `image`, `price`, `quantity`, `attributes`, `currency`)
- Replaced `Image` with `AuthenticatedImage` for authenticated image loading
- Simplified attribute rendering (removed color-swatch special case)

**PaymentPendingPageSection**
- Refactored to accept `orderId` as a prop and fetch payments via `getPaymentsByOrder`
- Removed all mock/sessionStorage data fallbacks
- Replaced product item preview and courier/address rows with a clean payment summary table (order number, amount, processing fee, payment method, status, phone)
- Control number card now only renders for `CONTROL_NUMBER` payment method
- Added skeleton loading state
- Fixed "View my orders" navigation to `/account/orders`

**Checkout — Delivery Selection**
- Added `isDeliveryConfirmed` state; validation now requires both a selected courier and a confirmed delivery API call before proceeding
- Replaced `setSelectedCourierId` direct setter with `handleCourierSelect`, which calls `selectDelivery` API, sets confirmation state, and shows success/error toast
- Wired `isSelecting` prop to `DeliveryCourier` to show a spinner on the active option and disable other options during the API call
- Fixed phone resolution to fall back to selected address phone before manual input
- Fixed post-payment redirect for control number / NBC account to include `orderId` in the path

**DeliveryCourier component**
- Added `isSelecting` prop; renders `Loader2` spinner on the selected option's icon while selecting, and dims/disables other options

**Routing**
- `/nbc-sokoni/cart/checkout/payment-pending` now redirects to `/account/orders` (no longer a standalone page)
- `/account/orders/[orderId]/page.tsx` now properly awaits async `params` and passes `orderId` to the section component
- Added dynamic route `/nbc-sokoni/cart/checkout/payment-pending/[orderId]` for payment pending with real order context

**Services & Providers**
- `checkoutService.selectDelivery`: new POST to `orders/{orderId}/delivery/select?deliveryMode={mode}`
- `paymentService.getPaymentsByOrder`: new GET to `payments/order/{orderId}`
- `CheckoutProvider`: added `isSelectingDelivery` state and `selectDelivery` action
- `PaymentProvider`: added `isFetchingOrderPayments`, `orderPayments` state and `getPaymentsByOrder` action

**Types**
- `checkout.ts`: added `DeliverySelectionItemI`, `SelectDeliveryRequestI`; added `isSelectingDelivery` and `selectDelivery` to `CheckoutContextI`
- `payment.ts`: added `PaymentTransactionI`, `OrderPaymentI`; added `isFetchingOrderPayments`, `orderPayments`, `getPaymentsByOrder` to `PaymentContextI`
